# WG-SRC / src_v16c 迭代思路一頁版

本文件用於快速閱讀。完整程式碼、結果和統計審計材料按時間順序保存在 `01_` 到 `16_` 各階段文件夾中。

## 一句話概括

本項目最初從 MCR² / ReduNet / CRATE 的白盒表示學習思想出發，嘗試解釋圖神經網絡內部機制；隨後發現訓練式 white-box GNN 仍不夠乾淨，於是轉向顯式圖信號、類 PCA 子空間、Fisher 坐標選擇和閉式 Ridge 分類，最終形成 WG-SRC：一個既能預測，又能生成數據及特徵的白盒圖分類器。

## 自己決定項目的項目發展方向的轉變
本項目的思路經歷了幾個階段：

1. 最初希望構建一個真正可解釋的 white-box GNN，減少傳統黑盒 GNN 中難以解釋的端到端訓練過程。

2. 隨後開始搭建白盒算法框架，並將圖分類過程拆解為圖信號構造、子空間表示、特徵選擇和閉式分類器等可測試模組。

3. 在實驗過程中，意識到只比較 accuracy 不足以解釋模型行為，因此進一步引入更細粒度的統計分析，觀察不同類別、特徵和數據集上發生了什麼。

4. 進一步地，發現該白盒框架不僅能用於分類，還可以刻畫數據集本身的結構與特徵模式，形成一種數據集特異性指紋分析。

5. 下一步，我計劃利用這些白盒統計結果，拆解傳統黑盒 GNN 在不同數據集上的表現差異，推斷不同黑盒部件再不通過數據集指紋之間的關聯，並據此繼續改進模型和實驗分析方法。


## 最終方法

最終版本是：

`14_src_v16c_FINAL_enhanced_multihop_Fisher_PCA_multi_alpha_Ridge`

核心組件：

1. 九塊顯式 graph-signal dictionary；
2. Fisher coordinate selection；
3. class-wise PCA residual；
4. multi-alpha closed-form Ridge；
5. validation-selected PCA/Ridge fusion；
6. node-level signal atlas。

## 關鍵科研轉折

### 轉折 1：從解釋黑盒到構造白盒

最初目標是用 MCR²、coding rate、effective rank、principal angle 等指標解釋模型內部表示。後來想嘗試將算法本身也白盒化。

對應目錄：

- `01_MCR2_layerwise_geometric_tracing`
- `02_whitebox_GNN_unfolding_before_G_series`

### 轉折 2：從 white-box GNN 層closed-form subspace classifier

G1 / CRGain 等 white-box GNN 路線曾經表現較強，但後續分析發現，僅僅給神經層加一個可解釋公式仍不夠乾淨。於是項目從“帶訓練層的 white-box GNN”轉向顯式矩陣運算和閉式分類器。

對應目錄：

- `03_G_series_CRgain_frequency_variational_rank_routes`
- `04_L_series_repairs_of_whitebox_GNN_route`
- `05_src_v1_layered_closed_form_whitebox_pipeline`

### 轉折 3：Chameleon 異配圖暴露 high-pass 圖信號的重要性，於是將白盒拆解為到可解釋性可拆分的相關部件，通過替換部件來驗證部件有效性

在 Chameleon 這類 heterophilic graph 上，單純 low-pass 平滑不夠。實驗發現 `X-PX`、`PX-P²X` 這類 high-pass graph difference 對分類非常關鍵。這一步直接導向最終九塊圖信號字典。

對應目錄：
- `05b_src_v3_to_src_v6_WhiteBox_GSD_pipeline_Layer3_coupling_and_packaging`
- `06_src_v7_Chameleon_highpass_Fisher_PCA_Ridge_breakthrough`

### 轉折 4：從 accuracy-driven 到更詳細的其他統計，比如pca錯誤點集特徵等

再迭代多次進步較小後，從v8 開始，迭代不再只是看 accuracy，而是看自主統計的：

- hard nodes；
- 參數對錯分節點的影響；
- branch fixed / broken / wrong redirect；
- per-class PCA dimension trace；
- pairwise specialist 是否只是 validation overfit；
- reliability map 是否真正提升 test transfer；
- OOF label beam 是否修得比破壞多。

對應目錄：

- `07_v8_error_node_and_parameter_effect_audit`
- `09_src_v10_safe_adaptive_branch_fixed_broken_redirect_audit`
- `11_src_v13_src_v14_per_class_dimension_coverage_and_floor`
- `12_src_v15_score_calibration_pairwise_specialist_validation_overfit`
- `13_src_v16_src_v16b_reliability_map_and_antihub`
- `15_rejected_after_v16c_v16d_v16e_v16f_v16g_v17`

### 轉折 5：最終選擇 src_v16c，因為它最乾淨，在預測能力與可解釋性上均表現優異，並且在最後開始轉向出個白盒特色結果的產生，比如數據集指紋

v16e、v17 等版本嘗試了 iterative labels、OOF label beam 等更複雜機制，但它們更慢、更複雜，且沒有穩定超過 src_v16c。最終選擇 src_v16c 是因為它在準確率、簡單性、白盒性、運行成本和 atlas 解釋能力之間最平衡。並得到不停數據集的是數據指紋，後續嘗試將指紋特徵與傳統黑盒模型部件構建相關聯繫

對應目錄：

- `14_src_v16c_FINAL_enhanced_multihop_Fisher_PCA_multi_alpha_Ridge`
- `16_FINAL_paper_experiment_package_and_six_dataset_results`

## 總結

該文件展示立項到完成科研的過程：

1. 從理論問題出發：MCR² / 子空間表示 / 白盒學習；
2. 發現方法內部問題：訓練式 white-box GNN 仍然不夠乾淨；
3. 根據失敗轉向：從 white-box GNN 轉向 closed-form classifier；
4. 針對數據特性改方法：heterophily 需要 high-pass graph signals；
5. 用統計審計做決策：fixed / broken / redirect、dimension trace、reliability map；
6. 排除複雜但不穩的方案：pairwise specialist、iterative label、OOF beam；
7. 最終形成可復現、可解釋、可診斷的算法框架 WG-SRC。
