# WG-SRC / src_v16c 迭代思路一页版

本文件用于快速阅读。完整代码、结果和统计审计材料按时间顺序保存在 `01_` 到 `16_` 各阶段文件夹中。

## 一句话概括

本项目最初从 MCR² / ReduNet / CRATE 的白盒表示学习思想出发，尝试解释图神经网络内部机制；随后发现训练式 white-box GNN 仍不够干净，于是转向显式图信号、类 PCA 子空间、Fisher 坐标选择和闭式 Ridge 分类，最终形成 WG-SRC：一个既能预测，又能生成数据及特征的白盒图分类器。

## 自己决定项目的项目发展方向的转变
本项目的思路经历了几个阶段：

1. 最初希望构建一个真正可解释的 white-box GNN，减少传统黑盒 GNN 中难以解释的端到端训练过程。

2. 随后开始搭建白盒算法框架，并将图分类过程拆解为图信号构造、子空间表示、特征选择和闭式分类器等可测试模块。

3. 在实验过程中，意识到只比较 accuracy 不足以解释模型行为，因此进一步引入更细粒度的统计分析，观察不同类别、特征和数据集上发生了什么。

4. 进一步地，发现该白盒框架不仅能用于分类，还可以刻画数据集本身的结构与特征模式，形成一种数据集特异性指纹分析。

5. 下一步，我计划利用这些白盒统计结果，拆解传统黑盒 GNN 在不同数据集上的表现差异，推断不同黑盒部件再不通过数据集指纹之间的关联，并据此继续改进模型和实验分析方法。


## 最终方法

最终版本是：

`14_src_v16c_FINAL_enhanced_multihop_Fisher_PCA_multi_alpha_Ridge`

核心组件：

1. 九块显式 graph-signal dictionary；
2. Fisher coordinate selection；
3. class-wise PCA residual；
4. multi-alpha closed-form Ridge；
5. validation-selected PCA/Ridge fusion；
6. node-level signal atlas。

## 关键科研转折

### 转折 1：从解释黑盒到构造白盒

最初目标是用 MCR²、coding rate、effective rank、principal angle 等指标解释模型内部表示。后来想尝试将算法本身也白盒化。

对应目录：

- `01_MCR2_layerwise_geometric_tracing`
- `02_whitebox_GNN_unfolding_before_G_series`

### 转折 2：从 white-box GNN 层closed-form subspace classifier

G1 / CRGain 等 white-box GNN 路线曾经表现较强，但后续分析发现，仅仅给神经层加一个可解释公式仍不够干净。于是项目从“带训练层的 white-box GNN”转向显式矩阵运算和闭式分类器。

对应目录：

- `03_G_series_CRgain_frequency_variational_rank_routes`
- `04_L_series_repairs_of_whitebox_GNN_route`
- `05_src_v1_layered_closed_form_whitebox_pipeline`

### 转折 3：Chameleon 异配图暴露 high-pass 图信号的重要性，于是将白盒拆解为到可解释性可拆分的相关部件，通过替换部件来验证部件有效性

在 Chameleon 这类 heterophilic graph 上，单纯 low-pass 平滑不够。实验发现 `X-PX`、`PX-P²X` 这类 high-pass graph difference 对分类非常关键。这一步直接导向最终九块图信号字典。

对应目录：
- `05b_src_v3_to_src_v6_WhiteBox_GSD_pipeline_Layer3_coupling_and_packaging`
- `06_src_v7_Chameleon_highpass_Fisher_PCA_Ridge_breakthrough`

### 转折 4：从 accuracy-driven 到更详细的其他统计，比如pca错误点集特征等

再迭代多次进步较小后，从v8 开始，迭代不再只是看 accuracy，而是看自主统计的：

- hard nodes；
- 参数对错分节点的影响；
- branch fixed / broken / wrong redirect；
- per-class PCA dimension trace；
- pairwise specialist 是否只是 validation overfit；
- reliability map 是否真正提升 test transfer；
- OOF label beam 是否修得比破坏多。

对应目录：

- `07_v8_error_node_and_parameter_effect_audit`
- `09_src_v10_safe_adaptive_branch_fixed_broken_redirect_audit`
- `11_src_v13_src_v14_per_class_dimension_coverage_and_floor`
- `12_src_v15_score_calibration_pairwise_specialist_validation_overfit`
- `13_src_v16_src_v16b_reliability_map_and_antihub`
- `15_rejected_after_v16c_v16d_v16e_v16f_v16g_v17`

### 转折 5：最终选择 src_v16c，因为它最干净，在预测能力与可解释性上均表现优异，并且在最后开始转向出个白盒特色结果的产生，比如数据集指纹

v16e、v17 等版本尝试了 iterative labels、OOF label beam 等更复杂机制，但它们更慢、更复杂，且没有稳定超过 src_v16c。最终选择 src_v16c 是因为它在准确率、简单性、白盒性、运行成本和 atlas 解释能力之间最平衡。并得到不停数据集的是数据指纹，后续尝试将指纹特征与传统黑盒模型部件构建相关联系

对应目录：

- `14_src_v16c_FINAL_enhanced_multihop_Fisher_PCA_multi_alpha_Ridge`
- `16_FINAL_paper_experiment_package_and_six_dataset_results`

## 总结

该文件展示立项到完成科研的过程：

1. 从理论问题出发：MCR² / 子空间表示 / 白盒学习；
2. 发现方法内部问题：训练式 white-box GNN 仍然不够干净；
3. 根据失败转向：从 white-box GNN 转向 closed-form classifier；
4. 针对数据特性改方法：heterophily 需要 high-pass graph signals；
5. 用统计审计做决策：fixed / broken / redirect、dimension trace、reliability map；
6. 排除复杂但不稳的方案：pairwise specialist、iterative label、OOF beam；
7. 最终形成可复现、可解释、可诊断的算法框架 WG-SRC。
