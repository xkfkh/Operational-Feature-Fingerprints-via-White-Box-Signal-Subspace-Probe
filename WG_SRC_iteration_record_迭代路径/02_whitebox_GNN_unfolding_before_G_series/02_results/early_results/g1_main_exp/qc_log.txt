## TASK1: Multi-dataset Generalization

### Bugs Found and Fixed
File: D:/桌面/MSR实验复现与创新/experiments_g1/task1_generalization.py

BUG 1 (CiteSeer - CRITICAL): load_citeseer used sp.vstack((allx, tx)) which
  produces wrong node count. CiteSeer test indices are non-contiguous and can
  exceed allx.shape[0]+tx.shape[0]-1. The original code then did
  features[test_idx_raw, :] = features[test_idx_sorted, :] which shuffled
  rows within the wrong-sized matrix, corrupting all test node features.
  FIX: Zero-pad feature matrix to num_nodes=max(max(test_idx_raw), allx+tx-1)+1,
       then place allx rows at [0:allx.shape[0]] and tx rows at exact test_idx
       positions using enumerate(test_idx_sorted).

BUG 2 (CiteSeer - CRITICAL): Row-normalization applied (lines 88-89 in old code).
  load_cora does NOT row-normalize. Inconsistent normalization causes the model
  to see a completely different feature distribution for CiteSeer vs Cora,
  leading to val=0.38, test=0.17 (near-random for 6 classes).
  FIX: Removed row-normalization from load_citeseer.

BUG 3 (PubMed - CRITICAL): Same row-normalization bug (lines 126-127 in old code).
  PubMed TF-IDF features should not be re-normalized. This caused val=0.437,
  test=0.407 (well below expected 0.75-0.82 range).
  FIX: Removed row-normalization from load_pubmed.

BUG 4 (CiteSeer val split): Old code returned range(200,500) = 300 nodes for val.
  Standard split is 500 val nodes. Fixed to range(200,700).

BUG 5 (PubMed val split): Old code returned range(200,500) = 300 nodes for val.
  Standard split is 500 val nodes. Fixed to range(200,700).

### Original (buggy) results:

  CiteSeer: val=0.380  test=0.166  (FAIL - expected 0.65-0.75)
